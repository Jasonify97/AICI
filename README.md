# <img src="https://github.com/Jasonify97/AICI/assets/98500133/604d456c-cd5d-46df-850b-884e713fe51c"  width="30" height="30"/>프로젝트 소개
👉[AICI 발표PPT.pdf](https://github.com/Jasonify97/AICI/files/12460207/36.3.36.pdf)


**본 프로젝트는 KT의 사업부서의 과제인 효율적인 업무를 위한 자동화 시스템 AICI로 사업 부서에서 제시한 2가지의 요구사항을 목적으로 함** </br>
* 고객TM 확인 서비스</br>
AICI 시스템으로 장애 영향 고객에게 동시다발 고객 TM을 시행 후 결과를 수합하여, 일괄적으로 확인, 관리할 수 있는 서비스 개발</br></br>
* 사외공사 신고 서비스</br>
공사 담당자가 AICI시스템으로 사외 공사를 신고하고, 관리 및 자동 응대를 시행할 수 있는 사외 공사 신고 서비스를 개발</br></br></br>

# <img src="https://github.com/Jasonify97/AICI/assets/98500133/324a82c7-c47a-40fa-b5fc-2b556b5f4da5"  width="30" height="30"/>프로젝트 목표
**1. 고객TM 확인 서비스**
* 장애 영향 고객에게 동시다발 고객 TM을 시행 후 결과를 수합
* 수합된 결과를 STT AI를 이용하여 양호/불량 판별
* AI를 이용해 텍스트 요약 후 긍부정 판별</br>

**2. 사외공사 신고 서비스**
* 사외공사 담당자의 전화를 받아 텍스트로 변환
* 텍스트에서 필요 단어를 추출 후 엔지니어에게 제공
* 공사 정보 캘린더 제작 및 공사 위치 제공</br>

**3. 기타**
* 공지사항 게시판
* 로그인 및 회원가입 페이지
* 개인정보처리방침/이용약관 페이지</br></br></br>

## 과제 배경
### **고객TM 확인 서비스**
 |<img width="600" alt="image" src="https://github.com/Jasonify97/AICI/assets/98500133/aa1ee977-671d-4b97-8cb4-ae1ed07447cf">|<img width="600" alt="image" src="https://github.com/Jasonify97/AICI/assets/98500133/095e46a5-067d-4328-bfda-a1d785287758">|<img width="600" alt="image" src="https://github.com/Jasonify97/AICI/assets/98500133/f67488ef-ca8e-4975-8ae8-bdfaee4d7d58">|
 |:---:|:---:|:---:|
* 현재 네트워크 장애 발생시 장애조치 이후 엔지니어가 고객에게 일일히 전화 후 장애가 해결 되었는지 확인하고 있음
* 하지만, 이는 엔지니어가 장애 조치 뿐만 아니라 전화까지 신경 써야 할 부분이 너무 많아져 장애 이장시간 및 VOC 발생 증가 가능성과 추가 장애 발생 가능성이 농후함
* 이를 해결하기 위해 AICI 시스템으로 장애영향 고객에게 동시 다발 TM서비스를 할 수 있도록 개발하고자 함
</br>

### **사외 공사 신고 서비스**
 |<img width="600" alt="image" src="https://github.com/Jasonify97/AICI/assets/98500133/7bc6164c-2f35-4607-9614-004f7d90979a">|<img width="600" alt="image" src="https://github.com/Jasonify97/AICI/assets/98500133/4fb456ec-34aa-4e0e-abde-924116de7ead">|<img width="600" alt="image" src="https://github.com/Jasonify97/AICI/assets/98500133/6247ada8-22bf-4f36-ae02-dc3c50369e11">|
 |:---:|:---:|:---:|
 * 사외 공사 신고의 경우 담당자가 담당자가 일일히 전화로 접수하고 있음
 * 이는 업무적으로 비효율적이며 취약시간의 경우 담당자의 부재일때 누락이 발생할 수 있음
 * 이로 인해 공사관리 미흡, 산업안전 리스크또한 발생할 수 있음
 * 이를 해결하기 위해 AICI시스템으로 사외 공사를 신고하고, 관리 및 자동응대를 시행할 수 있는 사외공사 신고 서비스를 개발하고자 함
</br>

## 과제 해석
### **고객TM 확인 서비스**
</br>
위 두가지 요구사항에 맞춰 과제를 수행하기 전 프로세스를 이해하고 과제를 해석
<p align="center"><img width="650" alt="image" src="https://github.com/Jasonify97/AICI/assets/98500133/844027e2-7209-4a15-bb2c-e81845d90d96"></p>

* 시스템에서 장애가 발생하면 관제 센터로 경보가 들어오며, 시스템을 사용중이던 이용자 또한 고객센터로 VOC가 들어옴
* 관제센터는 경보내용과 VOC 내용을 보고 엔지니어이게 수리 출동 요청 후 엔지니어에게 장애 해결 완료 보고를 받음
* 장애가 확실히 해결되었는지 판단하기 위해 신고가 접수 되었던 고객에게 확인 전화를 함
* 이에 저희는 VOC 내역들을 DB에 업로드 하며, 장애 영향 고객들에게 동시다발 TM서비스를 제공하고자 함
</br>
